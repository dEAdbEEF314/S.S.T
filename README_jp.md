# S.S.T (Steam Soundtrack Tagger)

S.S.T は、Steamで購入したサウンドトラックを自動的に識別・補完・タグ付けする、高精度な「ローカル完結型（エッジ処理）」システムです。Steam API、MusicBrainz、およびローカルの埋め込みタグから取得したメタデータを、LLM（Gemini等）を用いた「推論なしの事実整理」によって統合します。

## 🚀 システムアーキテクチャ: スタンドアロン・エッジ処理

S.S.T は従来の分散型アーキテクチャから、**スタンドアロン型のエッジ処理モデル**に移行しました。音声変換、LLMによる統合、タグ書き込みといった重い処理はすべて、Steamライブラリが存在するローカルマシン（WSL2/Windows等）上で行われます。これにより、ネットワークI/Oを最小限に抑え、プライバシーと制御性を最大化しています。

### コア・パイプライン
1.  **Scan**: ローカルキャッシュ（`scout_cache.json`）を活用したSteamライブラリのスキャン。
2.  **Enrich**: Steam API（親ゲーム情報の取得を含む）および MusicBrainz からのメタデータ取得。
3.  **Consolidate**: 1トラックずつの「対話型LLMセッション」によるメタデータの正規化（ハルシネーションの防止）。
4.  **Process**: 音声変換（ロスレス→AIFF、ロスレス以外→MP3）と、厳格なID3v2タグの書き込み。
5.  **Archive**: SeaweedFS (S3) へのアップロードと、`output/` ディレクトリへのローカルZIPパッケージの生成。

## ✨ 主な機能 (Act-10 アップデート)

-   **対話型 LLM チャット**: 曲数が多いアルバムでも、スライディングウィンドウ方式によるコンテキスト管理（TPM制限の遵守）を用いて、1トラックずつ確実に処理します。
-   **親ゲーム情報の統合**: サントラ側の情報が不足している場合、自動的に元となったゲーム本体（Parent Game）からタグやジャンルを取得します。
-   **厳格なメタデータ・フォールバック**: Steam/MBZ データを優先し、推論を排除した「真実」のみをタグ付けします。また、情報の起源を `source` フィールドに記録します。
-   **自動 ZIP パッケージ化**: 処理済みのアルバムを即座に利用可能な状態で `output/archive/` または `output/review/` に保存します。
-   **詳細なログ出力**: `llm_log.json`（全対話履歴）、`mbz_log.json`（生のAPIレスポンス）、`raw_metadata.json` を同梱し、透明性を確保。
-   **Web UI ダッシュボード**: 処理済みアルバムのブラウズ、メタデータの詳細確認、およびLLMの対話ログの確認が可能です。

## 🛠️ 技術スタック

-   **言語**: Python 3.12+
-   **パッケージ管理**: [uv](https://github.com/astral-sh/uv) (必須)
-   **ストレージ**: [SeaweedFS](https://github.com/seaweedfs/seaweedfs) (S3 + Filer API)
-   **AI**: OpenAI互換 LLM API (Gemini 等)
-   **音声処理**: FFmpeg, Mutagen
-   **フロントエンド**: React (Vite), Tailwind CSS, FastAPI (Backend proxy)
-   **実行環境**: Docker Compose (UI および ストレージ用)

## 🏗️ セットアップと実行

### 1. 準備
-   `.env` ファイルを設定してください（`.env.example` を参照）。
-   uv を使用して依存関係をインストールします：
    ```bash
    cd scout && uv sync
    ```

### 2. インフラ（ストレージ & UI）の起動
```bash
docker compose up -d
```

### 3. プロセッサの実行
```bash
# ルートディレクトリから scout の仮想環境を使用して実行
PYTHONPATH=scout/src ./scout/.venv/bin/python -m scout.main --limit 10
```

## ⚠️ 整合性とレビューのルール

-   **失敗の分離**: 必須タグ（タイトルまたはトラック番号）が不明なアルバムは、自動的に `review/` ステータスへ送られます。
-   **ハルシネーション・ゼロ**: LLMのプロンプトは推論を禁止するように設計されています。不明なデータは「Unknown」として残され、人間による確認を促します。
-   **タイムゾーン対応**: タイムスタンプは環境変数 `TZ` を参照し、ローカル時間で正確に記録されます。
