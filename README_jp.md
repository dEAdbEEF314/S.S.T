# S.S.T (Steam Soundtrack Tagger)

S.S.T は、Steamで購入したサウンドトラックを自動的に識別し、メタデータを補完してタグ付けを行う、高精度なスタンドアロンCLIツールです。Steam API、MusicBrainz、およびローカルの埋め込みタグからの情報を、LLM（大規模言語モデル）を用いた「事実に基づくメタデータ整理」によって統合します。

## 🚀 システムアーキテクチャ: スタンドアロン・エッジ処理

S.S.T は **ローカル完結型エッジプロセッサ** です。音声変換、LLMによる統合、タグ付けを含むすべての重い処理は、Steamライブラリが存在するローカルマシン（WSL2/Windows等）上で実行されます。これにより、ネットワークI/Oを最小限に抑え、プライバシーと制御を最大限に確保します。

### コア・パイプライン
1.  **スキャン**: Steamライブラリをスキャンし、ローカルのSQLiteデータベースを参照して処理済みアルバムを特定します。
2.  **情報補完**: Steam API（親ゲーム情報の統合を含む）および MusicBrainz からメタデータを取得します。
3.  **統合 (2段階LLM)**: 
    - **サマリー・パス**: アルバム全体のルール（アーティスト、ジャンル、ディスク構成）を決定し、一貫性を確保します。
    - **逐次パス**: サマリーで決めたルールに基づき、TPM制限を回避しながら各トラックのメタデータを統合します。
4.  **処理**: 音声を変換（Lossless -> AIFF, Lossy -> MP3）し、厳格な ID3v2.3 タグを書き込みます。
5.  **パッケージ化**: 音声ファイルと詳細な処理ログをZIPアーカイブにまとめ、`output/` ディレクトリに出力します。

## ✨ 主な機能 (Act-10 アップデート)

-   **高精度なCLI**: パワーユーザー向けに設計されており、詳細な進捗表示とターミナル上での確認機能を提供します。
-   **TPM最適化LLM**: グローバルな方針決定パスを導入することで、トラックごとのトークン消費を抑えつつ、アルバム全体の一貫性を維持します。
-   **スマート・タイブレーク**: MusicBrainzにおいて「Digital Media」形式かつ「Bandcamp」以外のソースを厳格に優先し、ゴールドスタンダードなメタデータを確保します。
-   **ローカル状態管理**: SQLiteデータベースを使用してすべての処理履歴を追跡し、無駄なAPIコールを防止します。
-   **親ゲーム統合**: サウンドトラック自体の情報が不足している場合、メインゲームのタグやジャンルを自動的に取得します。
-   **詳細なロギング**: すべてのZIPパッケージに `llm_log.json`（全チャット履歴）、`mbz_log.json`（生のリサーチ結果）、`metadata.json` が同梱されます。

## 🛠️ 技術スタック

-   **言語**: Python 3.12+
-   **パッケージマネージャー**: [uv](https://github.com/astral-sh/uv) (必須)
-   **AI**: OpenAI互換LLM API (Gemini 1.5 Pro 推奨)
-   **音声処理**: FFmpeg, Mutagen
-   **データベース**: SQLite (ローカル状態追跡用)

## 🏗️ セットアップ

### 1. 事前準備
-   `.env` を設定してください（`.env.example` を参照）。
-   `uv` を使用して依存関係をインストールします：
    ```bash
    cd scout && uv sync
    ```

### 2. 実行
```bash
# ルートディレクトリから実行
PYTHONPATH=scout/src ./scout/.venv/bin/python -m scout.main --limit 10
```

## ⚠️ 整合性とレビューのルール

-   **失敗の隔離**: 必須タグ（タイトルやトラック番号）が欠落している、またはメタデータの競合があるアルバムは、自動的に `output/review/` に振り分けられます。
-   **ゼロ・ハルシネーション**: LLMプロンプトは推論を排除するように設計されています。不明なデータは手動レビューのために "Unknown" のまま保持されます。
-   **ローカルDB**: アーカイブされたメタデータは `sst_local_state.db` に保存されます。このDBに存在する `app_id` はデフォルトでスキップされます。
