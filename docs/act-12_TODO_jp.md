# act-12 TODO: リッチなCLIとモードベースの挙動実装

目的: フェーズ5（CLI UX）を実装し、`ENV_MODE` と `LOG_LEVEL` に基づくシステム挙動を厳格化することで、プロフェッショナルなユーザー体験を提供する。

## 1. 環境設定とロジックの調整 (優先度: 高)
- [ ] **動的なエラークリーンアップ**: `ENV_MODE=development` かつエラー発生時、デバッグを容易にするために `temp_output` ディレクトリを削除せずに残すよう `processor.py` を更新。
- [ ] **本番環境用フェイルセーフ**: `ENV_MODE=production` の場合、1つのアルバムで致命的なエラーが発生しても、後続のアルバム処理を止めずに続行するようにする（現在は一部の例外が上位まで波及する可能性がある）。
- [ ] **ログの自動ローテーション**: `production` モードでは、詳細な `INFO/DEBUG` ログを `logs/sst_run_YYYYMMDD.log` にリダイレクトし、標準出力をクリーンに保つ。

## 2. フェーズ 5: リッチなCLIの実装 (優先度: 最重要)
- [ ] **マルチプログレスバー**: `rich` ライブラリを使用して以下を表示：
    - アルバムスキャン全体の進捗。
    - 各アルバム内のトラック変換/タグ付けの進捗（`MAX_ENCODING_TASKS` と同期）。
- [ ] **サマリーダッシュボード**: 各実行の最後に、成功、レビュー待ち、エラーの件数を美しく表示。
