# テスト環境仕様書 (ローカル完結型アーキテクチャ)

## 概要
このドキュメントは、S.S.T (Steam Soundtrack Tagger) システムのテストに使用される環境とインフラストラクチャについて記述します。システムは「ローカル完結型（エッジ処理）」モードで動作し、音源の処理、タグ付け、LLMによる統合は、Steamライブラリにアクセス可能なマシン（WSL2）上で直接実行されます。

## インフラストラクチャの依存関係
システムは以下の外部サービスに依存しています。

- **ストレージ (S3/SeaweedFS)**: `http://swfs-s3.outergods.lan`
  - バケット: `sst-data`
  - 処理済みのアーカイブ (`archive/`) およびレビュー待ちキュー (`review/`) に使用されます。`ingest/` プレフィックスは廃止されました。
- **AI / LLM**: `https://generativelanguage.googleapis.com/v1beta/openai` (Gemini API)
  - メタデータの統合および「整理者（Organizer）」ロジックに使用されます。
  - `.env` ファイルに `LLM_API_KEY` が必要です。

## コンポーネント構成
システムは主に2つのインターフェースに集約されました。

### 1. Scout (ローカルプロセッサ)
- **役割**: Steamライブラリをスキャンし、最適な音源ファイルを採用し、ローカル（WSL2）でフォーマットを変換し、LLMでメタデータを統合し、タグを書き込み、最終結果をS3にアップロードします。
- **設定**: `.env` (統合版)
- **主な実行コマンド**: `cd scout && uv run -m scout.main`

### 2. UI ダッシュボード (Docker)
- **役割**: アーカイブされたアルバムのリアルタイム監視、メタデータ閲覧、ZIPダウンロードを提供します。
- **設定**: `.env` (統合版)
- **URL**: `http://localhost:8000`

## 本番テスト実行手順

本番テストはワークスペースのルートディレクトリで直接実施されます。

### 1. 環境の準備
すべてのコマンドは、ワークスペースのルートである `/home/sexyroot/AI_Base/WorkSpace/S.S.T/` から実行してください。

`uv` を使用して依存関係を最新の状態にします。
```bash
# ワークスペースのルートから実行
uv sync
```

### 2. 設定
ワークスペースのルートに `.env` ファイルが存在し、S3 および LLM の必要な認証情報が含まれていることを確認してください。
システムは `SST_WORKING_DIR` (例: `/home/sexyroot/sst-work`) を使用して、Steamライブラリを改変することなくコピー/変換作業を行います。

### 3. テストの実行
サウンドトラックを処理し、タグ付けロジックを検証するには：

```bash
# ローカルプロセッサの実行
export $(grep -v '^#' .env | xargs)
cd scout
uv run -m scout.main --limit 5
```

### 4. 検証
- **Web UI**: `http://localhost:8000` にアクセスし、「Archive」タブを確認します。
- **S3 Filer**: `archive/{app_id}/` に、1トラックにつき1つの音声ファイル（AIFFまたはMP3）と `metadata.json` のみが存在することを確認します。
- **ZIP ダウンロード**: UI から ZIP をダウンロードし、Mp3tag などの外部ツールを使用してタグの内容を検証します。
