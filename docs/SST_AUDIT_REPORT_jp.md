# SST コードベース監査レポート (SST.md 基準)

**日付**: 2026-04-27
**ステータス**: 要アクション

本書は、現在の実装と「唯一の真実」 (`docs/SST.md`) との間の矛盾点を特定したものです。

## 1. 特定された矛盾点 (実装 vs 仕様)

### 1.1 判定ゲートの閾値の不整合
- **仕様 (SST.md 2.2)**: スコア 80-90% は **`REVIEW`** でなければならない。95% 以上のみが **`ARCHIVE`** となる。
- **実装 (`processor.py`)**: 現在、`conf_score < 80` を Review 送りのトリガーとして使用している。
- **アクション**: 閾値を `conf_score < 95` に変更する。

### 1.2 "Dirty Tags" (番号混入タイトル) のガード
- **仕様 (SST.md 2.2)**: 番号混入タグは強制的に **`REVIEW`** としなければならない。
- **実装 (`processor.py`)**: 最終バリデーションループにおいて、タイトルに番号が含まれているかどうかの明示的なプログラムチェックが行われていない。
- **アクション**: バリデーションループに正規表現チェックを実装：タイトルが `\d+ - ` などで始まる場合、強制的に **`REVIEW`** とする。

### 1.3 音源警告の強制適用
- **仕様 (SST.md 3.1)**: FFmpeg のいかなる警告も、強制的に **`REVIEW`** としなければならない。
- **実装 (`processor.py`)**: 前回の更新時のインデントの問題により、実装が不完全または不安定になっている。
- **アクション**: `any_audio_warnings` が確実に `status = "review"` を設定するように修正する。

## 2. 不足している定義 (SST.md での微調整が必要)

### 2.1 親ゲーム情報のフォールバック
- **課題**: `parent_app_id` が存在しない場合、`COMM` (コメント) をどのように構成すべきか。
- **提案**: 親ゲームが存在しない場合は、サウンドトラック自身の AppID/名称を使用する。

### 2.2 リリース年の衝突解決
- **課題**: Steam (Locked) と MBZ (Base) の日付が大きく異なる場合。
- **提案**: Steam の日付を `LOCKED` とするが、その差が 2 年を超える場合は LLM に **`REVIEW`** を推奨させる。
