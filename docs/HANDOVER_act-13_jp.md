# S.S.T オンボーディング & 引き継ぎ資料: Act-13

**Steam Soundtrack Tagger (S.S.T)** プロジェクトへようこそ。あなたは外部通知とシステムの堅牢化に焦点を当てた **Act-13** の開始時点から参加します。

## 1. Act-12 完了後のプロジェクトの状態
S.S.T は現在、以下を備えた完全に機能するスタンドアロン CLI ツールです：
- **リッチな CLI**: マルチプログレスバー追跡とカラーコード化されたロギング。
- **バッチ化された LLM 処理**: 大規模なコンテキストと出力制限を処理するためのステートフルな 2 フェーズ統合（Soul & Body）。
- **堅牢なスキャナー**: 隠れた DLC サントラのサポート。
- **起動スクリプト**: ルートの `./sst` を使用して、すべて（統計、ログ監視、実行、DB削除）を管理できます。

## 2. あなたのミッション: Act-13
主な目的は、**Discord Webhook 連携** を実装することです。

### 主要な技術領域:
- **`scout/src/scout/notify.py`**: (新規作成) リッチな埋め込みメッセージを含む Discord メッセージを送信するためのロジック。
- **設定**: 複数の Webhook レベル（Critical、Warning、Info、Completion）のサポート。
- **スロットリング**: エラーの急増時に Discord をスパムしないようにするためのロジック（クールダウン）。

## 3. 重要なファイル
- **`scout/src/scout/main.py`**: メインのオーケストレーションと CLI セットアップ。
- **`scout/src/scout/processor.py`**: 処理イベントが発生するコアロジック。
- **`sst`**: ルートの起動スクリプト。

## 4. 現在のワークフロー
- **実行**: `./sst -n 5`
- **統計**: `./sst --stats`
- **ログ監視**: `./sst --tail`
