# S.S.T 音源選択とメタデータ付与のルール

このドキュメントは、Steam Soundtrack Tagger (S.S.T) が各トラックに最適な音源ファイルをどのように選択し、複数のソースからメタデータをどのように統合して高精度なタグ付けを実現するのか、その理論的基盤と具体的なロジックをまとめたものです。このアプローチは、ネットワークのオーバーヘッドを排除するための「ローカル完結型（エッジ処理）」アーキテクチャに依存しています。

## 1. 基本理念

1.  **関心の分離:** どの音源ファイルを保持するかを選択するプロセス（ファイルの採用）と、タグを収集するプロセス（メタデータの収集）は完全に分離されます。
2.  **データを無駄にしない:** より高品質なファイル（例: FLAC）を採用するために、ある音源ファイル（例: 低品質なMP3）が破棄されたとしても、そのファイルに埋め込まれているメタデータは抽出され、貴重な情報源として活用されます。
3.  **LLMは「整理者」であり「創造者」ではない:** LLMは、提供された事実ベースのソースに基づいて、データの統合と競合の解決のみに使用されます。メタデータを推論したり、捏造（ハルシネーション）したりすることは厳しく禁じられます。
4.  **WAVのための再生時間ベースのマッチング:** 信頼できるメタデータを持たないWAVファイルは、正確な再生時間を計算し、信頼できる外部データベース（MusicBrainz）の再生時間と照合することで同定されます。
5.  **安全なフォールバック (Reviewモード):** LLMが確信を持って競合を解決できない、または情報が不足している場合、アルバムは安全に `review/` へ移動され、ユーザーの事後作業を支援するためにすべての生データソースとLLMのログが同梱されます。
6.  **外部アクセスの最小化:** 一度アーカイブされたアルバムのメタデータはローカルデータベースにキャッシュされます。処理済みのアルバムは、強制実行されない限り厳格にスキップされます。

---

## 2. ファイル採用ロジック（音源の選択）

1つのサウンドトラックに複数の音声フォーマット（同じアルバムのFLAC版とMP3版など）が含まれている場合、S.S.Tは論理トラックごとに最適な1つのファイルだけを採用しなければなりません。

### 2.1. 優先順位
システムは各論理トラックについて利用可能な全ファイルを評価し、以下の厳格な優先順位に従って最適なものを選択します。

1.  **ロスレス (最高優先度)**
    *   **ソース:** FLAC, WAV, ALAC など。
    *   **アクション:** **AIFF** に変換。
    *   **制限:** 最大24-bit深度、最大48kHzサンプリングレート。
2.  **高品質ロッシー (中優先度)**
    *   **ソース:** AAC, OGG, M4A (ALAC以外) など。
    *   **アクション:** **MP3** に変換。
    *   **制限:** 固定ビットレート(CBR) 320kbps、最大48kHzサンプリングレート。
3.  **標準ロッシー (最低優先度)**
    *   **ソース:** MP3。
    *   **アクション:** 変換を行わず **そのまま採用 (パススルー)**。MP3からMP3への再エンコードは音質向上を伴わずにファイルサイズを肥大化させるだけであるため、厳格に回避します。

### 2.2. 重複の破棄
最適なファイルが選択されると、その特定トラックの他のフォーマット・バリアントはすべて破棄されます。

## 3. メタデータ収集と例外処理

### 3.1. ソースの集約
1.  **フォーマット別のタグセット:** ファイルが破棄される前に *すべての* 音源ファイルから抽出されます。
2.  **Steam APIデータ:** Steamストアから直接アプリの情報を取得します。
3.  **MusicBrainzデータ:** Steamのアルバムタイトル（`.acf` の `name`）を使用して検索します。
    *   *タイブレークのルール:* スコア1位の候補が複数ある場合、以下の順で評価します：
        a. Format = Digital Media である
        b. "Bandcamp" の文字列が含まれていない
        c. トラック数(Tracks)が実際の音源ファイル数と同じか、より近い
        d. リリース日(Date)がSteamと同じか、より近い
4.  **ファイル名とパスのパース:** 内部タグが欠落しているトラックにおいて極めて重要です。ディレクトリ階層が複雑なケースもパース時に考慮されます。
    *   *注:* AcoustID はSteamのVGMにおけるヒット率が極めて低いため、意図的に除外または優先度を下げています。

### 3.2. WAVの例外処理（再生時間マッチング）
最高優先度のファイルがWAVである場合：
1.  正確な再生時間を計算します。
2.  MusicBrainzのトラックリストの再生時間と照合します。
3.  一致した場合、そのトラックのMusicBrainzのソースを「高信頼度」に引き上げてからLLMに渡します。

## 4. LLMベースの統合（真実の整理整頓）

生データは最終統合のためにLLMに渡されます。

### 4.1. LLMへのプロンプト指示
*   **ハルシネーションの絶対禁止:** メタデータをでっち上げてはならない。
*   **言語優先度 (TLAN):** 複数言語が存在する場合は、「ユーザー設定言語 ＞ 英語 ＞ 原語」の優先順位に従って選択する。
*   **競合の解決:** 提供された事実ソースのみに依存して、フォーマットや文字化けのエラーを修正する。

### 4.2. ターゲットとなるメタデータマッピング仕様
LLMは、整理された真実を以下のID3v2/標準タグにマッピングしなければなりません：

*   **TIT2 (Title):** MusicBrainzとファイル名からのオリジナル・トラック・タイトル（トラックごとに最も詳細な仕様を採用）。
*   **TPE1 (Artist):** 元データ、Steam、またはMusicBrainzからの作曲者名。
*   **TALB (Album):** `.acf` ファイルの `name` 文字列そのまま。
*   **TPE2 (Album Artist):** `[Developer] | [Publisher]` の形式。
*   **TCON (Genre):** `STEAM VGM, [Original Game Genre]` の形式（フィルタリングのため "STEAM VGM" は必須）。
*   **TIT1 (Grouping):** `[Series or Game Title] | Steam` の形式。
*   **COMM (Comment):** Steamからの `[Game Title] | [Original Game Steam Tags] | [Original Game AppID] | [URL]` の形式。
*   **TCOM (Composer):** SteamおよびMusicBrainzからの個人/ユニット名。
*   **TDRC (Year):** リリース年 (YYYY)。
*   **TXXX (MusicBrainzAlbumID):** マッチした場合のID。
*   **TRCK (Track Number):** 必須。
*   **TPOS (Disc Number):** 必須。`Disc/Total` の形式（例: 1枚のみの場合は `1/1`）。
*   **TLAN (Language):** 言語優先度のルールに基づく。

### 4.3. トラックごとのアートワーク
アートワークは、アルバム単位の代表画像だけでなく、トラックごとに異なるアートワークを持つサウンドトラックを尊重するため、*トラック単位* で評価・埋め込みが行われます。
*   **アートワーク仕様:** PNG形式、500x500ピクセル。正方形でない画像は黒背景でパディングする。

## 5. ワークフロー実行と状態管理

1.  **状態キャッシュ:** `archive/` 行きとして確定したアルバムのメタデータは、Steamの `app_id` をインデックスとしてローカルのデータベースに保管されます。
2.  **スキップロジック:** 一度処理した（`archive/` や `review/` に存在する）アルバムは、外部アクセスを控えるために、特別な `--force` オプションがない限り厳格にスキップされます。
3.  **Reviewワークフロー (人間の手によるフォールバック):** LLMが自信を持ってメタデータを確定できない場合、アルバムは `review/` に送られます。
    *   ユーザーは、採用された音源ファイル、`.acf` ファイル、収集したすべての生メタデータソースのJSON、および **LLMとのやり取りの全ログ** をまとめたZIPをダウンロードします。
    *   ユーザーはこれらの同梱された情報を元に、Mp3tagアプリやVGMdbなどを活用して手動でタグ付けを完了させます。
