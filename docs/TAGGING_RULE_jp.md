# SST 音源選択およびメタデータ・タグ付けルール

このドキュメントは、Steam Soundtrack Tagger (SST) における音源ファイルの選択、ティア分類、およびメタデータ・タグ付けの正式なロジックを定義します。

## 1. ファイル選択とティア・ロジック

SST は、Steam のサウンドトラック・ディレクトリで見つかったオーディオ・ファイルを論理トラックごとにグループ化し、利用可能な最高品質のバージョンを採用します。

### 1.1 品質ティア (Quality Tiers)
- **Lossless (ティア 1)**: FLAC, WAV, AIFF, ALAC
- **Lossy High (ティア 2)**: OGG, AAC, M4A
- **MP3 (ティア 3)**: 標準的な MP3 ファイル

同じトラックに対して複数のフォーマットが存在する場合、変換およびタグ付けのために最も高いティアのフォーマットが選択されます。

## 2. メタデータ同定ロジック

### 2.1 ハイブリッド統合 (Act-11)
本システムは、確定的なプログラム・ロジックと LLM（大規模言語モデル）による推論を組み合わせたハイブリッド・アプローチを採用しています。

1.  **プログラムによる前処理**:
    - **ローカル相互検証 (Cross-Validation)**: 異なるフォーマット（例：MP3 と FLAC）のタグが完全に一致する場合、「強い証拠 (Strong Evidence)」としてマークされます。
    - **MusicBrainz スコアリング**: タイトルの類似性、デジタル・メディアかどうか、およびトラック数の一致に基づいて候補をランク付けします。
2.  **LLM による統合**:
    - アルバム全体を 1 回のリクエストで一括処理します (All-in-One)。
    - LLM は、証拠の重みに基づいて、Steam、MusicBrainz、およびローカル・タグの間の矛盾を解決します。

### 2.2 MusicBrainz 候補のランク付けルール (a-f)
MusicBrainz の候補は以下の基準でスコアリングされます。
- **a. タイトル一致**: アルバム名が Steam の名称と一致 (+40)
- **b. フォーマット**: "Digital Media" を優先 (+50)
- **c. 文脈**: タイトルに "Bandcamp" を含まない (+20)
- **d. トラック数**: ローカルファイル数と完全一致 (+40)
- **e. 日付**: Steam の発売日と一致 (+40)
- **f. 親作品の日付**: 親ゲームの発売日と一致 (+30)

## 3. ID3v2.3 タグ付け標準

Windows や様々なメディア・プレーヤーとの互換性を確保するため、以下の厳格なフォーマットを適用します。

### 3.1 フィールド・マッピングとフォーマット
| ID3 フレーム | フィールド | フォーマット / ルール |
| :--- | :--- | :--- |
| **TIT2** | 曲名 | 統合された曲名。 "Unknown" は禁止。 |
| **TPE1** | アーティスト | 主な作曲者または演奏者。 |
| **TALB** | アルバム名 | Steam 公式のアルバム・タイトル。 |
| **TPE2** | アルバム・アーティスト | `[開発元] | [パブリッシャー]` |
| **TCON** | ジャンル | 接頭辞: `STEAM VGM, [ジャンル]` |
| **TDRC** | 年 | 発売年 (YYYY) |
| **TRCK** | トラック番号 | 単一の整数 (例: `1`, `16`)。 `n/N` 形式は使用しない。 |
| **TPOS** | ディスク番号 | `n/N` 形式 (例: `1/1`)。 1枚組の場合は必ず `1/1`。 |
| **TIT1** | グルーピング | `[ゲーム名] | Steam` |
| **COMM** | コメント | `[ゲーム名] | [タグ] | [AppID] | [ストア URL]` |
| **TLAN** | 言語 | ISO 639-2 コード (例: `jpn`, `eng`) |

### 3.2 技術的要件
- **エンコーディング**: すべてのテキスト・フレームで **UTF-16 with BOM (encoding=1)** を使用。
- **アートワーク**: 正確に 500x500 の PNG。正方形の画像は強制リサイズ、非正方形の画像は黒枠パディング。

## 4. 振り分けとバリデーション (Archive vs. Review)

### 4.1 Archive (承認)
以下の条件をすべて満たした場合のみ、アルバムは `archive/` に移動されます。
- LLM 自信度スコアが **80点以上**。
- どのフィールドにも "Unknown" が含まれていない。
- トラック番号が "0" ではない。

### 4.2 Review (要確認)
以下の場合はアルバムは `review/` に移動されます。
- LLM 自信度が低い (80点未満)。
- 必須メタデータが欠落している。
- ローカル・ファイルと MusicBrainz の間で深刻な矛盾がある。
