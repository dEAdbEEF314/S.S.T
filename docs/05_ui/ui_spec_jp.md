# UI仕様: S.S.T Web インターフェース

## アーキテクチャ
- **バックエンド/BFF**: FastAPI (Python)
  - S3 (SeaweedFS) へのプロキシとして機能します。
  - AppID ではなく「アルバム名」でのトラック集約を担当します。
  - ZIPファイルの動的生成（またはキャッシュ）を行います。
- **フロントエンド**: Vue.js または FastHTML（軽量かつ高速開発用）。
- **デプロイ**: `docker-compose` によって新設される `ui` コンテナとして稼働します。

## 状態管理と安全ルール（厳守）
**重要**: 処理が未完了のアルバムは絶対に表示・ダウンロードさせてはなりません。
アルバムが「準備完了（Ready）」と判定される条件：
1. `archive/` または `review/` に存在する処理済みトラックの合計数が、Scoutが初期JSONで生成した `track_count` と完全に一致すること。
2. 該当AppIDのPrefectタスクが現在実行中でないこと。

## ページ 1: アーカイブ (処理成功)
対象: `archive/` ディレクトリ.

### リストビュー
- **アルバム名** でグループ化。
- カラム:
  - アルバム名
  - トラック数
  - フォーマット (例: AIFF, MP3)
  - 処理完了日時
- アクション: 
  - `[+] 詳細を展開`
  - `[ZIPで一括ダウンロード]`

### 詳細ビュー (展開時)
- 付与されたメタデータ（タイトル、アーティスト、ジャンル、年など）のトラック一覧テーブル。
- 埋め込まれた 500x500 カバーアートのプレビュー表示。

## ページ 2: レビュー (要手動確認)
対象: `review/` ディレクトリ.

### ワークフローと目的
MP3Tagなどの外部ツールを用いた手動タグ付けを支援するための情報を提供します。ユーザーはZIPをダウンロードし、ローカルでタグ付けを行い、手動で管理します。

### リストビュー
- **アルバム名** でグループ化。
- カラム:
  - アルバム名
  - トラック数
  - **レビュー理由** (例: 「メタデータ欠損」「クロスフォーマット不一致」など)。
- アクション:
  - `[+] 詳細を展開`
  - `[ZIPで一括ダウンロード]`

### 詳細ビュー (展開時) と アシスト情報
ユーザーの手動タグ付けを支援するため、システムが処理中に収集した以下の情報を提示します：
- **欠損データのハイライト**: どのフィールド（アーティスト、年など）が空であるかをハイライトしたテーブル。
- **Steam ストアデータ**:
  - `Developer` / `Publisher`
  - `Steam App URL` (別タブで開くクリック可能なリンク)。
- **MusicBrainz データ (低信頼度で見つかっていた場合)**:
  - `MBID`
  - `VGMdb URL` (別タブで開くクリック可能なリンク)。
- **参照推奨リンク・パネル**:
  - Steam ストアページへの直接リンク（別タブ）。
  - VGMdb への直接リンク（MB経由で見つかった場合、別タブ）。
