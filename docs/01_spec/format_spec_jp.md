# フォーマット変換仕様

## 変換マトリクス

| 入力フォーマット | ターゲット・フォーマット | ビットレート / 品質設定 |
| :--- | :--- | :--- |
| **ロスレス** (.flac, .wav, .aiff, .m4a-alac) | **.aiff** | 最大 48kHz, 24-bit PCM (高品質リサンプリング) |
| **MP3** (.mp3) | **.mp3** | パススルー (そのまま保持) |
| **その他の損失あり** (.ogg, .m4a-aac 等) | **.mp3** | CBR 320kbps, 最大 48kHz |

---

## 排他的選択ポリシー (単一ファイル出力)

最高品質のライブラリを保証し重複を避けるため、システムは以下の優先順位に従って、**1トラックにつき1つのファイル**のみを出力しなければなりません。

1. **AIFF (ロスレスから)**: 最優先。トラックにロスレス音源が存在する場合、AIFF に変換し、他のフォーマットは無視する。
2. **MP3 (オリジナル)**: 第2優先。ロスレス音源はないが MP3 が存在する場合、オリジナルの MP3 を使用する。
3. **MP3 (変換)**: 最終手段。他の損失ありフォーマット（OGG 等）しか存在しない場合、320kbps の MP3 に変換する。

---

## オーディオ制約

- **サンプルレート**: 最大 **48,000 Hz**。
    - すべての変換において、ダウンサンプリング時の忠実度を最大化するため **`soxr`** リサンプラー（SoX 品質と同等）を使用します。
- **ビット深度**: 
    - AIFF の場合: **24-bit PCM** (`pcm_s24be`)。
- **MP3 エンコーディング**: 変換時は **CBR (固定ビットレート) 320kbps** を使用すること。

---

## 出力構造

`archive/{AppID}/{Disc}/{filename}.{ext}`  
*(ここで {ext} は 'aiff' または 'mp3' のいずれか)*

---

## 実装ルール

- 各ユニークなトラックに対して、「最高品質」の候補のみを処理する。
- タグ付けステージの前に、低品質な重複ファイルは削除または無視する。
