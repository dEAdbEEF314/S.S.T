# LLM戦略

## 主要な役割

- フォーマットの調整および正規化のみ（例：言語の標準化、"- OST"などの接尾辞の削除、大文字小文字の補正）。
- **重要：LLMは不足しているメタデータ（アーティスト、年、タイトルなど）を推測したり捏造したりしてはなりません。データが不足している場合、LLMはそれを `review/` 対象としてフラグを立てる必要があります。**

---

## デュアルプロバイダー

- プライマリ（主要）
- セカンダリ（補助）

---

## 比較

- 類似度 ≥ 0.8 → 承認
- 信頼度 ≥ 0.65 → 承認
- < 0.6 → レビュー

---

## 失敗時のハンドリング

- 片方が失敗 → もう片方にフォールバック（ペナルティあり）
- 両方が失敗 → ルールベースのフォールバック
- 必要なデータが不足 → レビューへ送る（ハルシネーションは許可されない）

---

## 対象

- タイトルの正規化
- アーティストの正規化
- アルバムの正規化
