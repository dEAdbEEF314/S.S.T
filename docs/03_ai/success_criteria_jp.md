# 処理の成功および振り分け基準 (Act-11)

このドキュメントでは、サウンドトラックの処理結果を `archive/`（承認）または `review/`（要確認）ディレクトリに自動的に振り分けるための条件を定義します。

## 1. Archive（承認）の条件

以下のメタデータ整合性チェックをすべてクリアした場合のみ、アルバムは自動的に `archive/` に移動されます。

1.  **高い自信度スコア**: 統合結果に対する LLM の自信度スコアが **80 以上**（100点満点）であること。
2.  **完全性**:
    - すべてのトラックに空ではない `TIT2`（曲名）があること。
    - すべてのトラックに正の整数である有効な `TRCK`（トラック番号）があること。
3.  **禁止されている値**:
    - メタデータのどのフィールド（曲名、アーティスト、アルバム名など）にも "Unknown" という文字列が含まれていないこと。
    - トラック番号が "0" ではないこと。
4.  **フォーマット**:
    - ジャンルが `STEAM VGM, ` で始まっていること。
    - ディスク番号が `n/N` 形式に従っていること。
5.  **技術的成功**: ファイル変換、画像処理 (500x500 PNG)、および ID3v2.3 タグ付けが例外なく完了していること。

## 2. Review（要確認）の条件

以下のトリガーのいずれかが発生した場合、アルバムは `review/` に送られます。

1.  **低自信度**:
    - LLM 自信度スコアが **80 未満** である。
    - 不確実性の理由は `llm_log.json` の `confidence_reason` に記録されます。
2.  **メタデータの欠落**:
    - 必須フィールドが欠落している、あるいは "Unknown" となった。
    - トラック番号が "0" と判定された。
3.  **スキーマの不一致**: LLM の出力が有効な JSON としてパースできなかった、あるいは要求されたトラック・マッピングと一致しない。
4.  **ハードウェア/API エラー**: API のタイムアウト、またはローカル処理エラー（FFmpeg など）。

## 3. 処理後の監査 (Audit)

ユーザーは `audit_archives.py` ユーティリティを使用して `archive/` ディレクトリの一括監査を行い、自動フィルタをすり抜けた「偽陽性」（例：でっち上げられた曲名）がないかを確認できます。
