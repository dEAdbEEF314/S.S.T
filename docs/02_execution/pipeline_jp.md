# 処理パイプライン

## 標準フロー

1. Steamメタデータの取得
2. 音声埋め込みメタデータ（カバーアート含む）の抽出
3. クロスフォーマット・バリデーション（複数フォーマットが存在する場合、アルバム単位のメタデータを比較）
4. MusicBrainz検索（主要な外部データベース）
5. 候補のフィルタリングとスコアリング
6. ソースの決定（既存の検証済みメタデータ > MusicBrainz > Steam）
7. AcoustIDへのフォールバック（最終手段のみ）
8. LLMによるメタデータ正規化（厳格なフォーマット調整のみ、**推測禁止**）
9. フォーマット変換
10. タグ書き込み
11. 出力の保存

---

## ファストトラック（高速処理）

- 既存メタデータのクロスフォーマット・バリデーションで高い信頼度が得られた場合、MusicBrainz/AcoustIDをスキップ。

---

## 失敗時のハンドリング

- データ不足 / 低信頼度 → レビュー（**厳守：不足しているタグを推測したり捏造したりしないこと**）
- エラー → リトライまたは失敗
