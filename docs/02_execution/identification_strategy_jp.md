# 識別戦略

## ソースの優先順位

1. 手動（Manual）
2. 既存のメタデータ（クロスフォーマット検証済み）
3. MusicBrainz
4. Steam
5. AcoustID（最終手段のフォールバック）

---

## クロスフォーマット・バリデーション（複数形式間検証）

- 同じアルバムに対して複数のフォーマット（例：FLAC, MP3）が存在する場合、それぞれのアルバム全体のメタデータとカバーアートを抽出します。
- 抽出されたメタデータをフォーマット間で比較します。
- 完全に一致する場合、その既存メタデータを極めて信頼性が高いものとして扱い、採用優先度を高めます。

---

## 候補の条件

- フォーマット = Digital Media
- トラック数 ±1
- タイトルの類似度 ≥ 0.80

---

## スコアリング

- metadata_match_score（クロスフォーマットの一貫性）
- title_similarity（タイトル類似度）
- track_count_match（トラック数一致）
- release_date_match（リリース日一致）

---

## 決定および「推測の厳禁」ルール

- ≥ 0.75 → 承認
- 0.70–0.75 → 検証
- < 0.70 → レビュー
- **重要**: 必要なメタデータが不足しており、信頼できるソースから確定できない場合、システムは必ずファイルを `review/` に送らなければなりません。**プロジェクト全体において、メタデータの推測や捏造は厳禁です。**
