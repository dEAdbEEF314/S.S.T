# S.S.T (Steam Soundtrack Tagger) 技術引き継ぎドキュメント

## 1. プロジェクト概要
S.S.T は、Steam上のサウンドトラックに高精度なメタデータを自動付与し、DJ機材互換のクリーンなファイルとしてアーカイブするローカルCLIツールである。
「一度のアーカイブで、一生の信頼を」を理念とし、疑わしいデータはすべて `Review` 判定とし、`Archive` には100%の信頼性を担保する。

## 2. 現在のアーキテクチャ (Ultimate Data Mode)
HTMLスクレイピングを全廃し、以下の「3層APIアーキテクチャ」によりメタデータを収集・統合している。

*   **階層1 (Official Store API)**: `appdetails` エンドポイントから日本語のアルバム名、公式ジャンル、リリース日を取得。
*   **階層2 (Local PICS Bridge)**: ローカルのDockerコンテナ (`steamcmd/api` @ 8080) を経由し、Steamの内部PICSデータベースから「構造化されたトラックリスト」「多言語クレジット」「レーベル情報 (`TPUB`)」を確実に取得。
*   **階層3 (Steam Web API)**: `STEAM_WEB_API_KEY` を用いて `IStoreBrowseService` を叩き、コミュニティが生成した「人気ユーザータグ」を取得（Cloudflareブロック回避のためSteamSpyから移行）。

## 3. 重要な技術仕様と決定事項

### メタデータ・タギング (TAGGING_RULE.md)
*   **ID3v2.3 の強制**: MP3ファイルはDJ機材との互換性のため厳格に ID3v2.3 で保存（年は `TYER` を使用）。
*   **COMMタグの仕様**: `[親ゲーム名], [人気タグ], [AppID], [ストア URL]` のカンマ区切りフォーマット。2000文字のサイズ上限を超える場合は、末尾からタグ単位で自動プルーニングを行う。

### 同定と判定ロジック (SST.md)
*   **決定論的ファストトラック**: MusicBrainz(MBZ)にSteamへの直接リンクがあり、かつMBZ/PICS/ローカル間でトラック数が完璧に一致する場合、LLMをバイパスして即座に `Archive` を確定する。
*   **安全ゲート**: `--all` (全件処理), `--delete-db`, `--finalize` の3つの重要コマンドには、誤爆防止のための「3段階確認プロンプト」を実装済み。

### パッケージングとログ出力
*   **バッファ分離**: 音声変換時の一時ファイルは `/tmp/sst-work/buffer_*` で行い、最終パッケージ (`output/`) には混入させない。
*   **人間可読ログの分離**: LLMとやり取りする生JSON（`mbz_log.json`, `metadata.json`）は出力先の `json/` サブディレクトリに隔離。人間がレビューしやすいよう、Markdownで綺麗にフォーマットされた `LLM_PROMPT.md`, `LLM_RESPONSE.md`, `METADATA.md` をルートに出力する。
*   **メインログ**: 通常は `SST_YYYYMMDD.log`、`--dev` 実行時は `SST_DEBUG_日時.log` として出力。`./sst --tail` で最新のログを自動追尾可能。

## 4. 実行環境とLLMモデル
*   **環境**: Windows 11 / WSL2 (Ubuntu) / Python 3.12 (`uv` 管理)
*   **LLMモデル**: ローカルの Ollama 上でビルドした **`qwen2.5-7b-sst`** を使用。`temperature 0` と専用システムプロンプト（監査官ペルソナ、ハルシネーション禁止）がモデルファイルに焼き込まれている。
