# UI実装 TODOリスト

## 1. 指摘されたUIの不備（要検証）
- [ ] **UI反映の問題**: ビルド後もブラウザに最新の「リスト表示」が反映されない。（キャッシュまたはマウントの遅延の可能性）。
- [ ] **コントロールの可視性**: 「一括削除用チェックボックス」と「メタデータ閲覧ボタン（SearchCodeアイコン）」が行内に明確に表示されていることを確認。
- [ ] **情報の常時表示**: アルバムタイトル等の重要情報が、マウスオーバーなしで常に表示されていることを確認（リスト表示への移行要件）。

## 2. 機能的な不備
- [ ] **ZIP内の音源ファイル不在**: ZIP生成ロジックは修正済みだが、そもそも S3 の `archive/` プレフィックス内に音源ファイル（.aiff/.mp3）が存在していない。
  - *原因*: Workerサービスが音源のアップロードに失敗しているか、不適切なパスを使用している可能性が高い。
  - *対策*: Workerの `upload_result` および `FFmpeg` 変換ステップをデバッグする。

## 3. 最終確認が必要な高度な機能
- [ ] **一括削除**: S3上の複数プレフィックスの削除がエンドツーエンドで動作することを確認。
- [ ] **メタデータ・インスペクター**: オーバーレイ/モーダルで整形されたJSONが正しく表示されることを確認。
- [ ] **再処理 (Retry)**: アルバムを `ingest/` に戻し、Prefect フローが再トリガーされることを確認。
- [ ] **手動承認**: `review/` から `archive/` への移動が正常に動作することを確認。

## 4. 技術的負債・修正事項
- [ ] **ファイルフォーマットの厳格化**: 仕様（ロスレス由来AIFF > オリジナルMP3 > 変換MP3 320kbps）の優先順位を再確認し、徹底する。
- [ ] **ファイル名のサニタイズ**: ダウンロードされる ZIP ファイル名が各OSで安全に使用できることを保証する。
- [ ] **ストレージの同期**: Workerのアップロード修正後、初期5件のアルバムを再処理してデータを揃える。
