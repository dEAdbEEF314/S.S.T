# sst-analyzer

このスキルは、直近のバッチ実行ログ（データベースの処理履歴）を解析し、システムがどのような判断を下したかを検証・分類します。主にテスト実行後の結果確認や、Archive/Reviewへの振り分けロジックが意図通りに機能しているかを証明するために使用します。

## 🛠 動作仕様
- `data/sst_local_state.db` の `processed_albums` テーブルを読み取ります。
- 指定された期間（デフォルトは直近24時間）のデータを集計します。
- **Review送りの原因分類**:
  1. LLM Phase 2 タイムアウト（エラー・不達）
  2. LLM Phase 1 低信頼度（スコア90未満による早期リジェクト）
  3. Validator降格（スコアは高いが、物理的な整合性チェックで問題が検出されたもの）
- **Archiveの妥当性検証**:
  - スコア95未満の不当なArchiveが存在しないか確認。
  - STEAM-TRUST機能によって救済されたアイテム数のカウント。

## 🚀 実行手順
分析を実行するには、以下のスクリプトを呼び出してください。

```bash
uv run python <path-to-skill>/scripts/analyze_batch.py
```

オプション引数：
- `--since-hours <N>`: 過去N時間分のデータを分析します（デフォルト: 24）。
- `--db <path>`: データベースファイルのパスを指定します（デフォルト: `data/sst_local_state.db`）。

実行後、コンソールにMarkdown形式の分析レポートが出力されます。必要に応じて、出力を `docs/` ディレクトリ等に保存してユーザーに報告してください。
