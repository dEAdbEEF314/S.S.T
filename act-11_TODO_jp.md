# act-11 TODO: ロジックの最適化

目的: 確定的なロジックをAIから分離し、「真実の固定 (Locking Truth)」を徹底することで、メタデータの精度を最大化し、LLMのトークン使用量を最小限に抑える。

## 1. 真実の固定とロジックの分離
- [ ] **Steamメタデータのハードロック**: 親ゲームのAppID、ゲーム名、ストアURL、デベロッパー、パブリッシャーは `processor.py` で直接適用し、これらのフィールドについてはLLMの判断を介さないようにする。
- [ ] **Ultra-Confirmedの優先適用**: MusicBrainzが `ultra_confirmed` の場合、トラック名とトラック番号のLLM統合をスキップし、MBZデータを直接採用するロジックの実装。
- [ ] **ソース優先度の強制**: ユーザー定義の `METADATA_SOURCE_PRIORITY` に基づき、LLMに投げる前にプログラム側でソースを事前フィルタリングする。

## 2. LLMの責務の洗練
- [ ] **コンテキストの軽量化**: URLやAppIDなどの確定的なフィールドをLLMプロンプトから削除。タイトル、アーティスト、アルバムのバリエーションのみを送信する。
- [ ] **競合時のみのLLM実行 (調査)**: ソース間に競合がある場合のみLLMを呼び出せるか？（MBZとSteamが完全に一致する場合はAIをスキップ）。
- [ ] **信頼度ベースのタグ付け**: LLMが低信頼度（または "Unknown"）を出力した場合、プログラムが最高優先度のソースロジックへ正しくフォールバックするようにする。

## 3. 信頼性とデータフロー
- [ ] **メタデータの後処理**: LLMの応答後にバリデーションレイヤーを追加し、すべての必須ID3タグがサニタイズ（UTF-8、長さ制限など）されていることを確認する。
- [ ] **洗練された競合レポート**: グローバル決定パスを強化し、ユーザーにとって読みやすい `conflict_report.json` を生成する。

## 4. メンテナンス
- [ ] ハイブリッドロジック（プログラム + AI）を反映するように `TAGGING_RULE.md` を更新する。
