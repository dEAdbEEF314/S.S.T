# S.S.T プロジェクト拡張計画 (2026/05/26 改訂)

## 1. [仕様化・実装済み] レガシーDJ機材向けバイリンガル・タイトル（Plan B）

### 1.1 背景と目的
古いCDJ等のレガシー機材における日本語文字化け問題を解決するため、タイトルタグに日本語と英語を併記する。

### 1.2 実装仕様 (2026/05/24 完了)
- **取得**: VGMdb(CDDB API), MBZ(Pseudo-Releases), Steam(l=en)から多言語情報を取得。
- **フォーマット**: `{local} / {en}`。同一時は単一表記。
- **制限**: 結合後60文字超過時は `{local}` のみを優先。
- **自動DiscID**: 再生時間からDiscIDを動的算出しVGMdbブリッジを確立済み。

## 2. [新規提案・設計中] AcoustID Release ID を軸とした「Steam決定論的」同定ロジック

### 2.1 目的
「アルバム名が微妙に異なるために検索に失敗する」ケースをゼロにし、音声指紋からボトムアップで直接「Steamで販売されているそのもの」を射抜く。

### 2.2 コア・アルゴリズム: 「全トラック共通 Release ID 方式」
1.  **AcoustID ボトムアップ同定**:
    - アルバムの全（または過半数）トラックに対して `meta="recordings+releases+releasegroups"` で一斉照合を行う。
    - 全トラックの検索結果に共通して含まれる **MusicBrainz Release ID (UUID)** を抽出する。
2.  **決定論的フィルタリング（Steam Anchors）**:
    - 特定された Release ID に対し、以下の「Steam決定論的」証拠を照合する。
        - **SteamDB Link**: MBZのリリースに当該 AppID のリンクが存在するか。
        - **Publisher Alignment**: SteamのパブリッシャーとMBZのレーベル名が一致するか。
        - **Format Priority**: `Digital Media` フォーマットを最優先。
3.  **位相（Topology）照合**:
    - Steamストアの公式トラックリスト（順序・曲数）と、MBZの該当リリースの構成を比較。

### 2.3 エディション選別と LLM 司法監査
Release ID が複数候補（通常盤、デラックス版、再発盤など）に残った場合、システムは以下の確定的情報を LLM に提供し、最終選別を委ねる。

- **提供情報**:
    - 各候補のタイトル、発売日、メディア、レーベル名。
    - Steamストアとのトラックリスト構成の差分（例: 「この候補にはボーナストラックが2曲多い」）。
    - 各候補を指し示した AcoustID の統計的一致率。
- **LLMの役割**: 
    - 「Steamでの販売形態（デジタル版、発売日）」というコンテキストに基づき、物理的構成が最も近い候補を一つに決定する。

### 2.4 実装タスク
- [x] `scout.ident.acoustid` のリクエスト拡張 (`meta` パラメータの追加)。
- [x] `scout.ident.mbz` に「Release ID からの直接フェッチ」機能を追加。
- [x] `scout.processor` に「Steamアンカー」による事前フィルタリングロジックを実装。
- [x] LLMプロンプトの更新（AcoustID統計情報とエディション比較コンテキストの注入）。

## 3. [新規提案・設計中] AcoustID Recording ID を活用した「曲単位」メタデータのハイブリッド注入

### 3.1 背景と目的
AcoustIDによるRelease IDの多数決（アルバム同定）が失敗するケース（1曲のみのミニアルバムや、複数アルバムに分散収録されている場合など）においても、AcoustIDが特定した**Recording ID（曲自体）**とそのメタデータ（曲名、アーティストなど）は非常に高い信頼性を持つ。
多数決から漏れた場合でも、これら「曲単位」の確定的証拠を破棄せず、検索ベースラインやLLMへのコンテキストとして活用することで、後続のテキスト検索精度とLLMの判断精度を底上げする。

### 3.2 実装仕様案
1. **曲単位の確定情報収集**:
   - `scout.processor` のAcoustIDサンプリングループ内で、Release IDの多数決用データに加えて、「トラックごとの確定タイトルやアーティスト情報」を収集・保持する。
2. **Local Baseline への注入**:
   - 収集したAcoustID由来のメタデータを `local_baseline["tracks"]` 等へ「物理的確定証拠」として注入する。
3. **MBZ検索およびLLM監査の強化**:
   - MBZ検索時、ローカルファイル名や埋め込みタグよりもAcoustID由来の確定タイトルを優先比較対象として採用する。
   - LLMに対して、「トラックNの正式タイトルはAcoustIDによってXであると証明されている」という強力なコンテキストを提供し、エディション選別の精度を向上させる。
