# 作業案プラン: レガシーDJ機材向けバイリンガル・タイトル（Plan B） [実装済み]

## 1. 背景と目的
レガシーなDJ機材（古いCDJ等）ではアルファベットと数字しか表示できず、日本語（2バイト文字）が全て「?」に文字化けしてしまう問題がある。これを解決するため、タイトルタグ（TIT2）にユーザー言語（日本語）と英語（ASCII文字）を結合して登録し、文字化け時でも曲の識別を可能にする。

## 2. 実装仕様 (2026/05/24 完了)

### 2.1 データの取得・生成
LLMに翻訳負荷をかけず、既存の公式データから抽出する。
- **VGMdb**: CDDB互換サーバーの言語別エンドポイントを活用し、公式な日本語および英語タイトルを直接取得。
- **MusicBrainz**: 「Pseudo-Releases」に含まれるローマ字（Romaji）や公式英訳データを活用。
- **Steam Store**: APIリクエストに `l=english` を付与し、英語の公式トラックリストを併用。

### 2.2 タグ書き込みロジック (`tagger.py` / `builder.py`)
- **結合フォーマット**: `{local} / {en}` （例: `決意 / Determination`）
- **重複回避**: `local` と `en` が同一の場合は単一表記とする。
- **文字数制限（Truncation）**:
  - 結合後のタイトルが **60文字** を超える場合は、視認性を優先して `{local}` のみを書き込むフェイルセーフを導入。

### 2.3 実装済みフェイルセーフ
- **60文字制限**: 結合後のタイトルが **60文字** を超える場合は、視認性を優先して `{local}` (日本語) のみを書き込むロジックを `builder.py` に搭載済み。
- **DiscID自動算出**: 物理CD情報がなくても、再生時間から DiscID を算出し VGMdb から公式日・英タイトルを取得するブリッジ機能を `vgmdb.py` に実装済み。

## 3. 並行実施タスク
- **LLMプロンプトの言語制約緩和**: 
  - 推論理由（`confidence_reason`等）はモデルにとって自然な言語（English推奨）を許容する。
  - メタデータ実体は `USER_LANGUAGE` を優先する。
  - 中国語（簡体字・繁体字）の混入防止策は維持する。
